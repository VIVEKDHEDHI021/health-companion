## 📏 Coding Rules

### General
- Use TypeScript strictly — no implicit `any` except where explicitly documented
- All service functions must check `supabase.auth.getUser()` before DB operations
- All native API calls must be wrapped in `if (Capacitor.isNativePlatform())`
- Use `cn()` from `@/frontend/lib/utils` for all Tailwind class merging
- Prefer `async/await` over `.then()` chains

### React Components
- Functional components only — no class components
- Props must be typed with explicit interfaces or type aliases
- Use `useEffect` cleanup to prevent memory leaks (cancel animation frames, clear intervals, unsubscribe)
- Dialogs use Radix `Dialog` primitive from `src/frontend/components/ui/dialog.tsx`
- Toast notifications use `sonner`: `import { toast } from "sonner"`

### Routing
- Routes are file-based in `src/routes/` via TanStack Router
- After adding a new route file, run `npm run dev` once — TanStack Router auto-generates `routeTree.gen.ts`
- Do NOT manually edit `routeTree.gen.ts`
- Protected pages go inside `src/routes/_authenticated/` folder

### Database
- All Supabase queries go through `src/db/client.ts` → imported as `import { supabase } from "@/db/client"`
- For server-only operations: `import { supabaseAdmin } from "@/db/client.server"`
- After adding a table, update `src/db/types.ts` manually with the new table Row/Insert/Update types
- Migrations are in `supabase/migrations/` — use timestamped filenames, never edit existing migrations

### Mobile
- Mobile-only code must check `Capacitor.isNativePlatform()` first
- Use safe area utilities: `pt-safe`, `pb-safe`, `mt-safe`, `mb-safe` CSS classes
- Dynamic imports for Capacitor plugins: `const { Camera } = await import("@capacitor/camera")`
- After any web build for mobile, run: `npm run build:mobile && npx cap sync android`

## 🏗️ Architecture Assumptions

1. **Supabase is the only backend** — no separate API server exists
2. **Auth is Supabase Auth only** — no JWT manipulation outside of Supabase session
3. **Row Level Security (RLS) is enforced** — all tables have RLS enabled, policies are user-scoped
4. **OCR is the smart scanner's core** — the scanner does NOT store images in Supabase (unless `image_url` is provided from external source)
5. **No global state manager** — state is managed locally via `useState` / React Context
6. **Mobile offline is additive** — web application has no offline behavior
