### ISSUE-001: `@capacitor-community/image-to-text` has no iOS SPM Package

| Field | Value |
|-------|-------|
| **Severity** | Low |
| **Priority** | Low |
| **Platform** | iOS |
| **Status** | Open |
| **Discovered** | 2026-06-26 |

**Description:**  
During `npx cap sync ios`, the following warning is emitted:
```
[warn] @capacitor-community/image-to-text does not have a Package.swift
[warn] Some installed packages are not compatible with SPM
```

**Impact:**  
On-device OCR via `@capacitor-community/image-to-text` will not work on iOS. All iOS OCR must go through the cloud-based pipeline (Cloud Vision / Gemini).

**Workaround:**  
The existing fallback chain already handles this — if on native iOS, use camera capture then call `/api/analyze-image` server endpoint. Local OCR is only used as final fallback on web.

**Resolution Plan:**  
Either find an alternative iOS SPM-compatible OCR plugin, or implement a native Swift OCR plugin using Apple's Vision framework.

### ISSUE-003: Release APK is unsigned — not suitable for Play Store

| Field | Value |
|-------|-------|
| **Severity** | High |
| **Priority** | High |
| **Platform** | Android |
| **Status** | Open |
| **Discovered** | 2026-06-26 |

**Description:**  
`assembleRelease` produces `app-release-unsigned.apk`. The Play Store requires a signed APK/AAB.

**Impact:**  
Cannot publish to Google Play Store without a keystore.

**Workaround:**  
Can install and test via `adb install app-release-unsigned.apk` (with USB debugging enabled).

**Resolution Plan:**  
1. Generate a keystore: `keytool -genkey -v -keystore glucolab.jks -alias glucolab -keyalg RSA -keysize 2048 -validity 10000`
2. Configure `android/app/build.gradle` with signing config
3. Add keystore to `local.properties` or CI environment secrets

### ISSUE-005: Manual `src/db/types.ts` requires sync after schema changes

| Field | Value |
|-------|-------|
| **Severity** | Medium |
| **Priority** | Medium |
| **Platform** | All |
| **Status** | Open |
| **Discovered** | 2026-06-26 |

**Description:**  
Supabase TypeScript types are manually maintained in `src/db/types.ts` instead of being generated by Supabase CLI. Adding a new table requires manual update.

**Impact:**  
Risk of type drift between actual database and TypeScript definitions. Could cause type errors if schema changes are not reflected.

**Workaround:**  
Manually add table definitions to `src/db/types.ts` after each migration.

**Resolution Plan:**  
Set up `supabase gen types typescript --project-id <id> > src/db/types.ts` in CI pipeline after each migration push.

### RESOLVED-001: White screen on Capacitor Android WebView

| Field | Value |
|-------|-------|
| **Severity** | Critical |
| **Priority** | Critical |
| **Platform** | Android, iOS |
| **Status** | ✅ Resolved |
| **Discovered** | 2026-06-25 |
| **Resolved** | 2026-06-26 |

**Description:**  
App showed a white screen on launch in the Capacitor WebView.

**Root Causes:**  
1. **Hydration Mismatch:** SSR hydration was failing because the WebView did not have a pre-rendered DOM to hydrate. TanStack Start's `hydrateRoot` expected a server-rendered DOM, but on native there is only a static HTML file.
2. **History Invariant Failure:** The router threw `Invariant failed` on startup because it default-configured browser URL path routing, which fails on custom local-server/file origins (e.g., `https://localhost/` on native web views).

**Resolution:**  
- Created `src/entry-client.tsx` with platform-aware mounting:
  - On native: `createRoot(rootEl)` — standard React SPA mount.
  - On web: `hydrateRoot(document)` — standard SSR hydration.
- Configured dynamic router history in `src/router.tsx`:
  - On native: Uses `createHashHistory()` to handle hash-based routing.
  - On web: Defaults to standard browser path-based history.
- Created `generate-index.mjs` to produce a proper SPA `index.html` for the Capacitor WebView.
